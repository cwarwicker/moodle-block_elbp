MIS Connections

If you have any problems or questions in setting up an MIS connection, please contact moodlesupport@bedford.ac.uk and we will be happy to assist you.



NOTE: All MIS connection types wrap table names and field names, so that in the unlikely event that you have tables or fields with spaces in, it doesn't break the query. This may mean that you need to supply case sensitive table and field names when using some database types, such as Firebird, Postgres, etc... So it is best to work on the assumption that all field and tables names will be case sensitive.


Examples



MySQL

For MySQL connections you can just supply the hostname/ip address in the host column, a connection string is not required.

Example database server details:
Server Name/IP Address: 11.22.33.44
Database Name: mydb
User: root
Password: mypass

  • Name - My Connection
  • Type - MySQL
  • Host - 11.22.33.44
  • DB User - root
  • DB Password - mypass
  • DB Name - mydb


SQL Server

For SQL Server connections you can specify either a connection string (varying depending on the drivers you are using), or a data source name if you have set one up on your server.

Example database server details:
(USING ODBC DRIVERS)
Server Name/IP Address: 11.22.33.44
Database Name: mydb
User: root
Password: mypass

  • Name - My Connection
  • Type - MSSQL
  • Host - Driver={SQL Server};Server=11.22.33.44,1433;Database=mydb (the 1433 is the port number)
  • DB User - root
  • DB Password - mypass
  • DB Name - (Not required as it is in the connection string)

Example database server details:
(USING SQLSRV DRIVERS)
Server Name & Instance/IP Address: localhost\SQLEXPRESS
Database Name: mydb
User: root
Password: mypass

  • Name - My Connection
  • Type - MSSQL
  • Host - localhost\SQLEXPRESS
  • DB User - root
  • DB Password - mypass
  • DB Name - mydb


Microsoft Access

For Microsoft Access connections you can specify either a connection string (varying depending on the drivers you are using), or a data source name if you have set one up on your server.

There are some limitations to MSAccess connections in our system, in that the database file will need to be in a network location that is accessible from your Moodle server, you cannot just point to another IP address for example.

Example database server details:
Location of database file on network: L:\Databases\Test\mydb.accdb
Database Name: mydb
User: root
Password: mypass

  • Name - My Connection
  • Type - MSAccess
  • Host - Driver={Microsoft Access Driver (*.mdb, *.accdb)};Dbq=L:\\Databases\\Test\\mydb.accdb;
  • DB User - root
  • DB Password - mypass
  • DB Name - (Not required as it is in the connection string)


PostgreSQL

For PostgreSQL connections you can just supply the hostname/ip address in the host column, a connection string is not required.

Example database server details:
Host/IP Address: localhost (same server as Moodle for this example)
Database Name: mydb
User: root
Password: mypass

  • Name - My Connection
  • Type - PostgreSQL
  • Host - localhost
  • DB User - root
  • DB Password - mypass
  • DB Name - mydb


Firebird

For Firebird connections you will need to specify a connection string to point towards the database file.

There are some limitations to Firebird connections in our system, in that the database file will need to be in a network location that is accessible from your Moodle server, you cannot just point to another IP address for example.

Example database server details:
Server Name/IP Address: localhost
Location of database file on network: L:\Databases\Test\mydb.fdb
User: SYSDBA
Password: masterkey

  • Name - My Connection
  • Type - Firebird
  • Host - L:\Databases\Test\mydb.fdb
  • DB User - SYSDBA
  • DB Password - masterkey
  • DB Name - localhost
